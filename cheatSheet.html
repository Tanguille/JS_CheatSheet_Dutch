Input

.value hoort enkel bij <input> elementen! checkbox / radiobutton = bool => element.checked 
Voorbeeld omgaan met radio buttons
HTML code:
<input type="radio" name="colors" value="red" checked>
<input type="radio" name="colors" value="green">
<input type="radio" name="colors "value="blue">

Javascript code: 
var checkedRadioButton = document.querySelector("input[name='colors']:checked")

<Select> Element

Properties:
  • .options: elementen uit Select, vormt array
    • .selected: bool die aangeeft of option geselecteerd is
    • .value
    • .text
  • .selectedIndex: index van eerste geselecteerde option (-1 = niets geselecteerd) => via dit zoek je geselecteerde optie op in .options array 
Voorbeeld omgaan met select elementen
HTML code:
<select id="selfruit" size="4"multiple>
    <option>appel</option>
    <option>peer</option>
    <option>banaan</option>
    <option>kiwi</option>
</select>

Javascript code:
let selectedOptions = document.querySelectorAll("#selFruit option:checked");

De variabele selectedOptions zal nu een lijstbevatten met de geselecteerde options.
Als je er maar eentje verwacht omdat het <select> element geen multiple vermeldt, kun je natuurlijk querySelector() gebruiken i.p.v. querySelectorAll().

Opgelet

  • .textContent en .innerHTML horen nooit bij <input> elementen. Strikt genomen, “bijna nooit” bv. wel als je de tekst op een button wil veranderen.
  • Om een een nieuw child element toe te voegen kun je beter .insertAdjacentHTML(positie, html) gebruiken (zie verderop) i.p.v. .innerHTML+= "..." te schrijven.
    positie: beforebegin, afterbegin, beforeend, afterend

Setup functie

const setup= () => {
    ...
}
            
window.addEventListener("load", setup);

De setup function is een event listener die eigenlijk de initialisatie code bevat van onze applicatie.
Typisch is dat hierin allerlei andere event listener functions aanhun resp. elementen worden gekoppeld.
Moet in meeste gevallen onderaan als laatste functie opgeroepen worden in JS.

Werken met meerdere events

let objecten = document.getElementsByClassName('object');
for (let i = 0; i < objecten.length; i++) {
    objecten[i].addEventListener('event', method);
}
            
Event.target: returned object waarvan event getriggered werd.
Event.currentTarget: returned object waarvan event momenteel uitgevoerd wordt.

Sort

Array.sort(sortFunctie): Sorteert array naar voorwaarden. 
Een array van getallen sorteren:
const compare= (a,b) => {
    return a-b;
}
let array=[34, 67, 12, 5, 23];
array.sort(compare);

Een array van strings sorteren:
const compare= (a, b) => {
    return a.localeCompare(b);
}
let array=["zebra", "aap", "giraf", "ezel"];
array.sort(compare);

Attributen van DOM-elementen manipuleren

Attributen zijn de parameters die in de html tag staan.
De waarde van attributen zijn altijd strings.
Je kan custom attributen maken door "data-kiesnaam" te gebruiken.

  • element.getAttribute(a): returned waarde van attribuut a van element. (a parameter = string met naam attribuut)
  • element.setAttribute(a, value): stelt attribuut a van element in op v.

We maken een onderscheid tussen methods die alle soorten nodes returnen en degene die elementen returnen
n staat voor node in deze context.

  • n.parentNode: Geeft parent, of null indien er geen is.
  • n.children (geeft enkel elementen) / n.childNodes: Geeft lijst van childs (geen array maar .length en index werken)
    • n.first(Element)Child: geeft eerste child
    • n.last(Element)Child: geeft laatste child
    n.firstElementChild en n.lastElementChild, komen vaak van pas bij het gebruik van insertAdjacentHTML() op posities"afterbegin" en "beforeend", als verwijzing naar het nieuwe element!
  • n.next(Element)Sibling / n.previous(Element)Sibling: Geeft de volgende/vorige sibling, of null indien er geen is.
  • n.nodeName:Geeft soort element als string
  • n.nodeType:Geeft soort element als getal (1=element, 2=attribuut, 3=text)
  • n.nodeValue: (enkel text nodes)Geeft tekst in string of null indien element node.
